# JYDB数据替换说明

## 当前口径

本目录用于生成和每日更新风险平价策略的日度收益输入数据。旧目录 `JYDB期货数据替换` 已改为 `JYDB数据替换`，旧文件名 `JYDB主力日涨跌幅_填充.csv`、`JYDB主力日涨跌幅_未填充.csv` 已分别改为 `日涨跌幅_填充.csv`、`日涨跌幅_未填充.csv`。

输出文件：

- `日涨跌幅_填充.csv`
- `日涨跌幅_未填充.csv`
- `期货主力前复权收盘价.csv`
- `日涨跌幅更新摘要.md`

本目录还维护 `增量缓存/`，用于保存从 JYDB 和 iFinD 拉取后的累计原始输入数据：

- `增量缓存/期货行情.csv`
- `增量缓存/红利低波ETF行情.csv`
- `增量缓存/GC001.csv`

## 输出列

保留较新策略版本需要的字段：

`沪深300主连`, `10年国债主连`, `沪金主连`, `豆粕主连`, `中证1000主连`, `30年国债主连`, `红利低波ETF`, `一天期国债逆回购`, `沪铜主连`, `沪铝主连`, `PTA主连`, `原油主连`

删除不再参与较新策略版本的数据列：

`有色ETF`, `能源化工ETF`, `布油连续`

## 数据源

- 期货主连：JYDB `dbo.Fut_TradingQuote`、`dbo.Fut_DailyQuote` 和 `dbo.Fut_ContractMain`。
- 红利低波ETF：JYDB `512890.SH`，`InnerCode = 201577`，从 `dbo.DZ_DailyQuote` 读取。
- 一天期国债逆回购：iFinD EDB `L004369613 / GC001(加权平均)`，单位 `%`。

JYDB 密码不写入本文档、SQL、CSV 或更新摘要。每日更新脚本会自动读取仓库根目录 `.env`，并从 `JYDB_PWD`、`JYDB_SERVER`、`JYDB_DATABASE`、`JYDB_UID`、`JYDB_DRIVER` 获取 JYDB 配置；iFinD MCP 优先读取 `IFIND_MCP_URL` 和 `IFIND_MCP_AUTHORIZATION`。如果 `.env` 或当前环境变量缺少 iFinD 配置，脚本会回退读取 `--ifind-config` 指定的 TOML，默认是 `~/.codex/config.toml`。

## 计算方法

- 期货主连先取 `MainContractMark = 1` 的每日主力合约收盘价，再按主力切换比例生成连续前复权收盘价。
- 期货日涨跌幅 = 前复权收盘价逐日百分比变化，单位为百分比点。
- `红利低波ETF` 日涨跌幅 = `ClosePrice / PrevClosePrice - 1`，再乘以 100。该口径可处理除权除息或拆分日。
- `一天期国债逆回购` 直接使用 iFinD 的 `GC001(加权平均)` 年化利率，不从 JYDB 的开高低收字段推算。

## 填充规则

- `日涨跌幅_未填充.csv` 只保留 JYDB 和 iFinD 可获得的数据，品种上市前或覆盖不足的区间保留为空。
- `日涨跌幅_填充.csv` 在未填充版基础上，用旧填充版历史数据补齐官方源不可得的早期缺口，保证历史回测连续。

## 每日更新

运行：

```powershell
python 数据\JYDB数据替换\update_daily_returns.py
```

默认流程：

1. 先读取本地 `增量缓存/` 的最新日期。
2. 只向 JYDB/iFinD 拉取缓存缺失的新日期，并默认向前覆盖 7 天以处理数据源修订或延迟。
3. 合并更新本地缓存，重叠日期以新拉取数据覆盖旧缓存。
4. 从完整本地缓存重新计算期货主力前复权价格和日涨跌幅，再全量覆盖输出 CSV。

这种方式减少重复访问数据库，同时保留当前前复权口径；主力切换时，最终输出仍由完整缓存重算，避免只追加输出行造成复权标尺不一致。

常用参数：

- `--end-date YYYY-MM-DD`：指定更新截止日期。
- `--cache-overlap-days N`：增量缓存更新时向前覆盖 `N` 天，默认 `7`。
- `--full-refresh`：忽略现有缓存，从数据源重新初始化缓存和输出。
- `--rebuild-from-cache`：不访问 JYDB 或 iFinD，只用现有缓存重算输出；缓存缺失时会报错。
- `--dry-run`：只查询和校验，不写文件。
- `--ifind-config PATH`：指定 iFinD MCP TOML 配置；当 `.env` 已提供 `IFIND_MCP_URL` 和 `IFIND_MCP_AUTHORIZATION` 时不会读取该文件。

如果缓存不存在，脚本会按历史起点初始化缓存。`--start-date` 仍表示输出起始日期；期货缓存初始化时会额外向前读取 `--lookback-days`，用于保证首日收益率和复权计算有足够上下文。
