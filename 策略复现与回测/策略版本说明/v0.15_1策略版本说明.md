# v0.15_1 策略代码说明

## 1. 文件定位

- 策略代码：`策略复现与回测/策略测试代码/资产风险平价策略0.15_1（decay参数优化测试）.py`
- 版本号：`0.15_1`
- 基础版本：`v0.15`
- 主要目标：在 v0.15 的 JYDB 主力未填充交易收益回测框架上，对 EWMA 半协方差模型中的 `decay` 参数做训练期选参和验证期检验。

## 2. 本轮测试设计

本轮不再直接用全样本选择参数，而是拆分为：

- 训练期：`2013-01-01` 至 `2020-12-31`，实际从策略首个可交易日开始。
- 验证期：`2021-01-01` 至数据末尾，当前数据截至 `2026-03-31`。
- 参数范围：`0.50, 0.53, ..., 0.98, 0.99`。
- 选择规则：仅使用训练期的 `全天候增强策略 (动量择时)` 指标选参；按夏普比率、年化收益、最大回撤依次排序。

对应代码配置：

```python
EWMA_DECAY = 0.50
DECAY_TEST_VALUES = [round(0.50 + i * 0.03, 2) for i in range(17)] + [0.99]
TRAIN_PERIOD_LABEL = '训练期 (2013-2020)'
VALIDATION_PERIOD_LABEL = '验证期 (2021-2026)'
```

## 3. 主要代码修改

1. `run_backtest` 在原有全局和年度指标外，新增训练期、验证期两个回测区间。
2. `decay参数优化测试结果_v0.15_1.csv` 现在包含全局、训练期、验证期三类结果。
3. 参数选择逻辑只读取训练期的 `全天候增强策略 (动量择时)`，验证期结果只用于样本外检查。
4. 当前训练期选择结果为 `decay=0.50`，因此新版默认 `EWMA_DECAY` 同步为 `0.50`。

## 6. 结论

按严格的训练期选参规则，当前版本选择 `decay=0.50`，并用该参数生成 v0.15_1 的净值、指标、仓位和图表。

但训练期和验证期存在一个重要差异：训练期偏好更低的 `decay`，验证期偏好中低区间的 `0.71-0.83`。这说明 `decay` 的最优值并不完全稳定。相比直接全样本选出的 `0.62`，训练期选参结果更接近一个样本外测试流程，但也提示未来可考虑采用更稳健的参数区间，例如 `0.65-0.80`，而不是只取单点最优。
